INCIDENT RESPONSE - DETECTION & ANALYSIS PHASE


Incident: Suspected Ransomware (CRITICAL)

---

1. INITIAL TRIAGE ASSESSMENT & SEVERITY VALIDATION

Severity Confirmation: CRITICAL - VALIDATED


  • Active ransomware deployment with visible ransom note

  • Business-critical asset (File Server) compromised

  • Data availability impact - encryption in progress or completed

  • Potential lateral movement risk to other systems


Immediate Status Classification


  • Incident Type: Confirmed Ransomware Attack

  • Attack Phase: Post-Encryption (ransom note visible)

  • Containment Priority: IMMEDIATE

  • Business Impact: HIGH (data unavailable, potential spread)


---

2. KEY INDICATORS OF COMPROMISE (IOCs) - IMMEDIATE COLLECTION

Priority 1: File System Indicators (Next 5 minutes)


  • Encrypted file extensions (e.g., .locked, .encrypted, .crypt, random extensions)

  • Ransom note filenames (README.txt, HOW_TO_DECRYPT.html, RECOVER_FILES.txt)

  • Mass file modifications - Check last modified timestamps for bulk changes

  • New executables in temp directories (%TEMP%, \Windows\Temp, C:\ProgramData)

  • Deleted shadow copies (vssadmin list shadows)


Priority 2: Network Indicators


  • Command & Control (C2) connections:

- Suspicious outbound connections on ports: 443, 8080, 4444, non-standard ports
- TOR network connections (.onion domains, known TOR exit nodes)
- Cryptocurrency wallet API connections
  • Lateral movement: SMB (445), RDP (3389), WinRM (5985/5986) connections

  • Data exfiltration: Large outbound data transfers (check NetFlow/firewall logs)


Priority 3: Execution Indicators


  • Suspicious processes:

- PowerShell with encoded commands (-enc, -EncodedCommand)
- wscript.exe, cscript.exe executing from unusual locations
- Recently created executable processes
- PsExec, WMI command line execution
  • Scheduled tasks or new services created within last 72 hours

  • Registry modifications: Run keys, service entries


Priority 4: Authentication Indicators


  • Failed/successful login attempts - especially Administrator/service accounts

  • Account creation or privilege escalation (last 72 hours)

  • RDP brute force attempts or suspicious successful authentications

  • VPN or remote access logs for unauthorized access


---

3. CRITICAL LOG SOURCES - CHECK IMMEDIATELY

Tier 1: File Server Logs (0-10 minutes)


``
Priority Order:
  • Windows Security Event Log (Security.evtx)

- Event IDs: 4624, 4625 (logon/failed), 4672 (special privileges), 4688 (process creation)

  • Windows System Event Log (System.evtx)

- Event IDs: 7045 (service installation), 7036 (service state change)

  • PowerShell Logs

- Microsoft-Windows-PowerShell/Operational
- Event IDs: 4103, 4104 (script block logging)

  • File System Audit Logs

- Event ID: 4663 (object access attempts), 4660 (object deletion)

  • Shadow Copy Service Logs

- Look for deletion events (vssadmin delete shadows)
`

Tier 2: Network Security Logs (10-15 minutes)


`
Priority Order:
  • Firewall logs - last 72 hours

- Outbound connections from file server
- Unusual port usage or destination IPs

  • IDS/IPS alerts

- Ransomware signatures, malware callbacks

  • DNS query logs

- C2 domain lookups, DGA patterns, TOR-related queries

  • Proxy/Web gateway logs

- File downloads, suspicious URLs accessed from file server

  • NetFlow/network traffic analysis

- Bandwidth spikes, unusual internal scanning
`

Tier 3: Endpoint Detection & Backup Logs


`
  • EDR/Antivirus logs - file server and connected systems

  • Backup system logs - verify last successful backup, check for backup deletion

  • SIEM correlation - cross-reference with other alerts

``

---

4. IMMEDIATE QUESTIONS - FIRST 15 MINUTES

Critical Operational Questions


  • When was the ransom note first observed? (Exact timestamp)

  • Is encryption still actively occurring? (Check CPU usage, disk I/O, file modification times)

  • What is the last known-good backup timestamp? Are backups accessible and unencrypted?

  • How many systems are affected? (Just file server or lateral spread?)

  • Is the file server still connected to the network? (Isolation status)


Technical Investigation Questions


  • What user account is currently logged in or was last active on the file server?

  • Are there any active suspicious processes running? (Check Task Manager/Process Explorer)

  • Have Volume Shadow Copies been deleted? (vssadmin list shadows)

  • What is the external IP connectivity? (netstat -ano for active connections)

  • Are domain controllers or backup systems communicating with the file server?


Scope Assessment Questions


  • What data/shares are hosted on this file server? (Classification: PII, financial, PHI?)

  • How many users/systems have mapped drives to this server?

  • Are there any recent security alerts or AV detections on this server? (past 7 days)

  • Has anyone received phishing emails recently? (potential initial access vector)

  • Were any legitimate user credentials used? (vs. exploitation)


---

5. IMMEDIATE NOTIFICATION REQUIREMENTS

Notify Within 5 Minutes (CRITICAL PATH)


  • Chief Information Security Officer (CISO)

- Critical severity incident declaration

  • IT Security Operations Manager
